Historique

Le langage C a été conçu en 1972 par Dennis Richie et Ken Thompson, au laboratoire Bell Labs, en vue de développer un nouveau système d'exploitation, Unix, portable sur différents hardwares. En 1978, Brian Kernigham et Dennis Richie publie une première définition du langage, The Programming Language, connue sous le nom C K&R ou C classique. Vu l'intérêt suscité par le langage, il est adapté par de nombreux développeurs et une standardisation s'avère nécessaire, les travaux entamés en 1983 dureront 6 années et déboucheront sur une nouvelle norme, l'ANSI C. En 1990, l'ISO adopte telle quelle la norme. D'autres évolutions ont suivis que le présent cours de considérera pas.

Avantages

Le langage C reste un langage des plus utilisés, vu ses très nombreux avantages dont les principaux sont :

langage structué de haut niveau
permet une programmation aisée générant un code très efficace car proche d'un langage d'assemblage, donc du langage machine.
La portabilité
Un programme écrit en ANSI C, pour un certain système peut facilement être déployé sur d'autres systèmes, Windows, Unix, Linux, Pc, mini, mainframe, etc
code performant
Peu de code "inutile", de vérifications, etc, car il est supposé que le programmeur sait ce qu'il écrit. De plus les types définis par le langage sont proches de la représentation internes des données.
Aisé à apprendre
Vu la simplicité du langage,il peut être facilement assimilé.

Inconvénients

Tous ces avantages impliquent certains inconvénients auxquels les programmeurs devront faire très attention : Les compilateurs C faisant peu de vérifications, le programmeur doit être attentif à sécuriser son code, adopter un style de programmation stricte, programmer de manière structurée et claire

1er programme

Voici un premier programme qui affiche un message de bienvenue sur stdout 

	1 #include <stdio.h>
        2
	3 int main(){
	4	printf("Hello world\n");
	5	return 0;
	6 }

Dans cet exemple, la ligne

  1. est une directive du préprocesseur qui inclut le contenu du fichier stdio.h dans notre source.
  2. est une ligne vide, ignorée par le compilateur
  3. est le nom de la fonction principale, point d'entrée de l'application.
  4. est l'appel de la fonction printf qui affiche sur stdout.
  5. est l'instruction qui permet de quitter l'application en renvoyant la valeur 0 au shell appelant.
  6. indique la fin de la fonction main.
Avant d'exécuter notre application, nous devons compiler la source, c-à-d la traduire en langage machine. La commande pour y arriver est la commande cc. 
 cc pgm1.c
où pgm1.c est le nom de la source. Cette commande, si il n'y a aucune erreur, va générer un programme exécutable, appelé par défaut a.out. Avec l'option -o, nous pouvons spécifier un autre nom pour notre exécutable. 
 cc -o pgm1 pgm1.c
